# 시니어 중심 모바일 설문 디자인 가이드

> 대상: 충남대학교 평생교육원 수강생(시니어 비중 높음) · 모바일 우선
> 목적: 누구나 설명 없이 끝까지 응답을 완료하도록 하는 디자인 기준 정의
> 한 문장 원칙: **"읽기 쉽고, 누르기 쉽고, 헷갈리지 않게."**

---

## 1. 설계 7원칙

1. **크게**: 글씨도 버튼도 시니어 기준으로 크게.
2. **한 번에 하나**: 한 화면에 한 가지 주제만 묻는다.
3. **분명한 대비**: 배경과 글자, 선택 전후 상태를 또렷이 구분한다.
4. **즉각적 피드백**: 누르면 바로 색이 변해 "선택됐다"는 것을 보여준다.
5. **항상 보이는 진행**: 지금 어디쯤인지 항상 위에 표시한다.
6. **쉬운 말**: 약어·전문용어 대신 일상어로 짧게.
7. **실수 방지**: 필수 누락 시 막기보다 "친절히 알려주고" 바로 고치게 한다.

---

## 2. 타이포그래피

| 요소 | 최소 크기 | 권장 | 비고 |
|------|-----------|------|------|
| 문항 텍스트 | 22px | 24~26px | 가장 중요한 글자 |
| 본문/안내문 | 18px | 20px | |
| 버튼 라벨 | 18px | 20px | 굵게(600) |
| 보조 안내(예시) | 16px | 17px | 회색, 단 대비 유지 |

- 줄간격(line-height) 1.5 이상으로 넉넉하게.
- 한 줄 글자 수를 줄여 읽기 부담을 낮춘다.
- 폰트: **Pretendard / 맑은 고딕 / 시스템 고딕** 등 가독성 높은 산세리프. 장식체 금지.
- 절대 단위보다 `rem` 기반 + 사용자 브라우저 확대 허용(텍스트 줌 차단 금지).

---

## 3. 색상·명도대비

- **명도대비**: 본문 텍스트 4.5:1 이상(WCAG AA), 핵심 버튼·문항은 7:1(AAA) 지향.
- **기본 팔레트(초안)**
  - 배경: `#FFFFFF` / 섹션 구분 `#F5F7FA`
  - 본문 글자: `#1F2937` (진한 회색, 순수 검정보다 눈 피로 적음)
  - 강조/주 버튼: 충남대 상징색 계열 단일 강조색 + 흰 글씨
  - 선택됨 상태: 강조색 채움 + 굵은 테두리
- 색만으로 의미를 전달하지 않는다(색 + 글자/아이콘 병행) — 색약 사용자 배려.

---

## 4. 터치 영역·간격

| 항목 | 기준 |
|------|------|
| 버튼/선택지 최소 높이 | 56px (권장 60px 이상) |
| 터치 영역 최소 너비 | 화면 폭에 맞춰 가능한 넓게(척도 버튼은 가로 꽉 차게 권장) |
| 요소 간 간격 | 최소 12px, 권장 16px 이상 |
| 좌우 여백 | 16~20px |
| 하단 [다음] 버튼 | 화면 하단 고정, 엄지로 누르기 쉬운 위치 |

---

## 5. 5점 척도 컴포넌트 (가장 중요)

시니어에게 가장 어려운 부분이므로 별도로 정의한다.

- **세로 배치 권장**: 작은 가로 5칸 라디오 대신, 위에서 아래로 **큰 버튼 5개**를 쌓는다(모바일에서 누르기 쉬움).
- 각 버튼에 **숫자 + 라벨 + 색/표정 보조**를 함께 표시.

```
┌───────────────────────────┐
│  ⑤  매우 만족        😀     │  ← 선택 시 채워짐
├───────────────────────────┤
│  ④  만족            🙂     │
├───────────────────────────┤
│  ③  보통            😐     │
├───────────────────────────┤
│  ②  불만족          🙁     │
├───────────────────────────┤
│  ①  매우 불만족      😟     │
└───────────────────────────┘
```

- D3·D4(의향 문항)는 라벨을 "전혀 없음 ~ 매우 있음"으로 교체.
- 선택 시: 배경 채움 + 테두리 강조 + 체크 표시로 명확히.
- 표정 이모지는 보조 수단일 뿐, 숫자·라벨이 항상 우선.

## 6. 입력 요소별 가이드

| 유형 | 디자인 |
|------|--------|
| 단답형(A1 강좌명) | 큰 입력칸(높이 56px+), 큰 글씨, 예시 placeholder 제공 |
| 객관식(A2·A3) | 세로로 쌓인 큰 버튼형 선택지(라디오보다 버튼 권장) |
| 복수선택(A4) | 큰 체크박스 카드, "여러 개 선택 가능" 안내 명시 |
| 척도(B·C·D) | 위 5점 척도 컴포넌트 |
| 장문형(E1~E4) | 큰 텍스트영역, "선택 사항" 명시, 음성입력 안내(선택) |

---

## 7. 진행·이동·피드백

- **진행 표시**: 상단에 막대 + "전체 5단계 중 2단계" 큰 글씨.
- **이동 버튼**: 하단에 [이전](보조) / [다음](주 강조). 색·크기로 위계 구분.
- **필수 검증**: 미응답 시 다음 단계 비활성 대신, 누르면 해당 문항으로 스크롤 + "이 문항에 답해 주세요" 큰 안내.
- **중간 저장**: 뒤로 가도 입력값 유지(이탈·재진입 대비).
- **제출 전 확인**: "제출하시겠습니까?" 큰 확인 화면.
- **완료 화면**: 큰 감사 메시지 + 체크 아이콘, 더 이상 조작 불필요.

---

## 8. 접근성 체크리스트

- [ ] 모든 텍스트 대비비 AA 이상
- [ ] 버튼/선택지 높이 56px 이상
- [ ] 브라우저 글자 확대(최대 200%)에도 레이아웃 깨지지 않음
- [ ] 색 없이도 의미 전달(라벨·아이콘 병행)
- [ ] 입력 요소에 명확한 라벨 연결(label/aria-label)
- [ ] 포커스 표시 명확(키보드 사용자)
- [ ] 이미지·아이콘에 대체 텍스트
- [ ] 저사양·느린 통신에서도 빠른 로딩

---

## 9. 하지 말 것 (시니어 사용성 저해 요소)

- 작은 글씨, 좁은 줄간격, 회색 위 회색 같은 낮은 대비.
- 한 화면에 너무 많은 문항을 몰아넣기.
- 가로로 빽빽한 라디오 버튼(오터치 유발).
- 자동 진행/시간 제한/팝업 광고성 요소.
- 마우스 호버에만 의존하는 정보(모바일에 없음).
- 어려운 한자어·영어 약어 남용.

---

## 10. 구현 메모

- Tailwind CSS 사용 시, 위 기준을 **디자인 토큰**으로 고정한다(예: `text-senior-base`, `btn-touch-lg`).
- 실제 화면 시안과 컴포넌트는 개발 단계에서 프론트엔드 디자인 작업으로 제작하며, 본 문서를 기준으로 한다.
